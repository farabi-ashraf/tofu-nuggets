## Standing policies (binding)

### B2 — one branch, ever

**No per-platform branches.** One codebase, one `main`; platform differences live behind
`#[cfg(...)]`/traits; per-platform installers come from a CI build matrix on the same
commit. Branch-per-platform means hand-porting every fix N times — forbidden.
When a second platform actually starts: extract a `DesktopIcons` trait
(icon_at / list_icons / selected_icon / suppress_tips) with the Win32 impl behind
`#[cfg(windows)]`. Do NOT abstract before then.

### Release process

1. Bump version in `app/src-tauri/tauri.conf.json` + `Cargo.toml` (+lock) via PR.
2. Tag the merge commit `vX.Y.Z`, push the tag → CI builds, signs, drafts the release
   with `latest.json`.
3. Owner reviews + publishes the draft (manual by design). Updater serves it from
   `releases/latest/download/latest.json`.

### Publish gate

Before any visibility change or anything history-related goes public: run the
history-scan checklist in `docs/SECURITY.md`.
